💻 Usage Examples

Basic Setup

<my-audio-player data-tracks='[ { "src": "./audio.mp3", "title": "Song Name", "duration": 240 } ]'></my-audio-player>

Accessing Components

const player = document.querySelector('my-audio-player'); const eq = player.shadowRoot.querySelector('#eq'); const playlist = player.shadowRoot.querySelector('#playlist'); const viz = player.shadowRoot.querySelector('#viz'); // Control playback player.playNext(); player.toggleShuffle(); player.toggleLoop(); // EQ operations eq.applyPreset('Bass Boost'); eq.toggleBypass();

Event Listening

// Listen for playlist changes playlist.addEventListener('play-track', (e) => { console.log('Playing track:', e.detail.index); }); // Listen for EQ changes eq.addEventListener('eq-change', (e) => { console.log(`Band ${e.detail.index}: ${e.detail.gain}dB`); }); // Standard audio events const audio = player.shadowRoot.querySelector('#myplayer'); audio.addEventListener('play', () => { viz.start(); });

🎛️ Component Architecture

The player uses a clean separation of concerns, with each component managing its own state and UI:

my-audio-player (Main) ├── my-playlist │ ├── Track list with drag-drop │ ├── Delete buttons │ └── Duration display ├── my-equalizer │ ├── 10 BiquadFilters │ ├── Preset selector │ ├── 10 gain sliders │ └── Bypass toggle └── my-visualizer ├── Canvas rendering ├── Spectrum/Waveform modes └── Real-time animation Web Audio Graph inside player: <audio> → Equalizer → GainNode → Panner → Analyser → Destination

Key Features by Component

  • my-audio-player: Orchestration, audio graph, controls UI
  • my-equalizer: 10-band processing, presets, bypass
  • my-visualizer: Real-time spectrum/waveform rendering
  • my-playlist: Track management, drag-reorder, auto-advance

🚀 Advanced Features

🔀 Shuffle Mode

Randomizes track order using Fisher-Yates algorithm. Auto-generates shuffled index array on toggle.

🔁 Loop Modes

  • Off: Play all tracks, stop at end
  • Loop All: Restart playlist after last track
  • Loop One: Repeat current track infinitely

🎛️ Web Audio Chain

Properly constructed audio graph with single pass through processing nodes. Supports real-time filter adjustments without artifacts.

📱 Responsive Design

Components adapt to different viewport sizes. Playlist, controls, and visualizer stack vertically on mobile.
